Schreibe mir ein Google Apps Script für Google Docs, das als zentrales Master-Knowledge-Dokument (Datenbasis für NotebookLM / Gemini) dient. Das Script soll es ermöglichen, Inhalte aus über 55 einzelnen Google Docs strukturiert in dieses eine Master-Dokument zu importieren und dort zentral zu verwalten.

Hier sind die genauen Anforderungen:

1. HTML-BENUTZEROBERFLÄCHE (Sidebar/Overlay):
- Über ein benutzerdefiniertes Menü in Google Docs soll sich eine Sidebar öffnen.
- Die UI benötigt ein einfaches Eingabefeld für einen Google Docs Link und einen "Importieren"-Button.
- Darunter soll eine Liste aller bereits importierten Dokumente angezeigt werden.
- Diese Liste muss durchsuchbar sein. Jedes Element in der Liste braucht zwei Buttons: "Anspringen" (Scrollt im Master-Doc zum Inhalt) und "Entfernen" (Löscht den Inhalt komplett aus dem Master-Doc).

2. IMPORT-LOGIK & FORMATIERUNG:
- Beim Import wird der Inhalt des verlinkten Dokuments geöffnet und kopiert.
- Wichtig: Formatierungen, Tabellen, Listen und Fettungen müssen exakt 1:1 beibehalten werden (Element- und Attribut-Kopie).
- Der Name des Quell-Dokuments definiert die Kategorie. Dieser Name wird als "Überschrift 1" im Master-Dokument über dem importierten Text eingefügt.
- Damit die Hierarchie für NotebookLM logisch bleibt, müssen alle vorhandenen Überschriften aus dem Quell-Dokument beim Import um eine Ebene nach unten verschoben werden (Quell-H1 wird im Master zu H2, Quell-H2 zu H3 usw.).

3. ARCHITEKTUR & VERWALTUNG (CRUD):
- Da ein Google Doc ein linearer Textfluss ist, muss das Script importierte Bereiche eindeutig identifizieren können. Nutze dafür "Named Ranges" (Benannte Bereiche), die auf der eindeutigen Dokument-ID des Quell-Docs basieren.
- Wenn ein Nutzer in der Sidebar auf "Entfernen" klickt, sucht das Script die entsprechende Named Range im Master-Dokument und löscht alle darin enthaltenen Elemente restlos.

Bitte liefere mir den vollständigen Code aufgeteilt in:
- Code.gs (Backend-Logik für Google Docs)
- Sidebar.html (Frontend für die Benutzeroberfläche mit sauberem, einfachen CSS)
